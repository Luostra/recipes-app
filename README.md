# Проект приложения на Flutter на тему "Кулинарные рецепты" (RecipesApp)
## Подготовил Ключников А.Д., ЭФБО-09-23

## **1. Введение**

Современные мобильные устройства стали универсальным инструментом для хранения информации, планирования и самоорганизации. Одной из популярных сфер применения мобильных приложений является кулинария — многие пользователи ищут рецепты, сохраняют их, структурируют по тематикам и используют в повседневной жизни.
Цель данного проекта — разработать мобильное приложение на Flutter для Android, позволяющее пользователю удобно хранить, упорядочивать и редактировать собственные кулинарные рецепты.

---

## **2. Анализ предметной области**

### **2.1 Понятие кулинарного рецепта**

Кулинарный рецепт — руководство по приготовлению кулинарного изделия. Содержит информацию о необходимых пищевых продуктах, их пропорциях и инструкциях по смешиванию и обработке. Кулинарные рецепты описывают механическую и тепловую обработку ингредиентов, способы сервировки готовых изделий.

В прошлом рецепты могли передаваться устно или фиксироваться в кулинарных книгах, но в наши дни цифровая форма стала стандартом хранения.

### **2.2 Проблемы пользователей в предметной области**

Пользователи, активно занимающиеся кулинарией, часто сталкиваются со следующими сложностями:

* рецепты хранятся в разных источниках (заметки, мессенджеры, бумажные записи), что приводит к утерям;
* отсутствует единая система классификации и удобный поиск;
* изображения блюд часто хранятся отдельно от текста;
* нет удобного способа быстро структурировать личную коллекцию рецептов.

---

## **3. Область применения**

Разрабатываемое приложение будет полезно:

* домашним поварам, хранящим собственные авторские рецепты;
* любителям кулинарии, регулярно готовящим новые блюда;
* пользователям, желающим структурировать свою коллекцию;
* тем, кто хочет систематизировать рецепты по тематическим папкам или категориям.

В рамках приложения рецепт определяется как информационная структура, включающая:

* название блюда;
* описание (шаги приготовления, продукты, заметки) в свободном формате;
* изображение готового блюда.

---

## **4. Существующие решения**

На рынке доступны различные приложения, связанные с кулинарией:

* **Cookpad** — кулинарная социальная сеть с рецептами пользователей;
* **Allrecipes** — крупная платформа с профессиональными и любительскими рецептами;
* **Kitchen Stories** — приложение с редакционными рецептами и видеогидами.

Однако перечисленные продукты имеют существенные ограничения для пользователей, заинтересованных именно в личной структурированной коллекции:

* невозможно создать полную собственную систему папок;
* присутствуют реклама и социальные элементы, мешающие минималистичному использованию;
* отсутствует гибкий механизм управления собственными рецептами.

## **5. Пользователи и их потребности**

Целевая аудитория приложения:

* любители кулинарии;
* пользователи, создающие авторские рецепты;
* люди, предпочитающие хранить свою коллекцию локально и структурировано;
* владельцы домашних рецептов, ранее записанных на бумаге или в заметках.

Основные потребности пользователей:

* быстрое создание рецептов;
* возможность добавлять фотографии;
* удобная классификация по папкам;
* редактирование и удаление записей;
* избранные рецепты для быстрого доступа;
* простой, интуитивный интерфейс без лишних элементов.

---

## **6. Общее описание приложения**

Планируемое мобильное приложение представляет собой персональный каталог кулинарных рецептов.
Основные компоненты интерфейса:

* экран авторизации (регистрация / вход);
* список папок с рецептами;
* отдельная закреплённая папка «Избранное»;
* экран с рецептами конкретной папки;
* просмотр и редактирование рецепта;
* добавление изображений (с камеры или галереи) с обязательным кадрированием под квадрат.

Пользователь после авторизации создаёт собственные тематические папки, каждая из которых содержит рецепты. Внутри рецепта пользователь может указать название, описание и прикрепить фото.
Система позволяет помечать рецепты как избранные, что делает доступ к ним моментальным.

## **7. Планируемый функционал**

### **7.1 Авторизация**

* регистрация через email + пароль;
* вход / выход из аккаунта;
* защита пользовательских данных посредством RLS.

### **7.2 Управление папками**

* создание, редактирование и удаление папок;
* задание названия, описания и emoji-иконки;
* автоматическое создание системной папки «Избранное», которую нельзя изменить или удалить;
* отображение «Избранного» в верхней части списка.

### **7.3 Управление рецептами**

* добавление рецепта в выбранную папку;
* ввод названия и свободного текстового описания;
* прикрепление изображения;
* редактирование содержимого;
* удаление рецепта;
* добавление/удаление из избранного через отметку-звёздочку;
* автоматическое отображение избранных рецептов в системной папке.

### **7.4 Работа с фотографиями**

* выбор фото из галереи;
* создание фото с помощью камеры;
* обязательное кадрирование под квадрат;
* загрузка изображений в приватное облачное хранилище;
* ограничение доступа благодаря политике безопасности хранилища.

---

## **8. Логика поведения системы**

### **8.1 При запуске**

1. Проверяется, авторизован ли пользователь.
2. Если да — открывается список папок.
3. Если нет — отображается экран входа/регистрации.

### **8.2 При создании папки**

1. Пользователь вводит название и описание.
2. Приложение отправляет запрос в базу данных.
3. Созданная папка появляется в списке.

### **8.3 При добавлении рецепта**

1. Открывается форма с названием, описанием и кнопкой добавления фото.
2. Пользователь добавляет изображение, оно кадрируется.
3. Изображение отправляется в облачное хранилище.
4. Сохраняется запись рецепта в БД.

### **8.4 При добавлении в избранное**

* поле `is_favour` переключается в `true`;
* рецепт появляется в папке «Избранное».

### **8.5 При удалении рецепта**

* из обычной папки — удаляется полностью;
* из «Избранного» — снимается флаг избранного, но запись остаётся в изначальной папке.

## **9. Описание уровней, архитектуры и сервисов**

### **9.1 Архитектурные слои**

Предполагается реализация следующей архитектуры:

#### **1) UI слой (Flutter Widgets)**

* отображение элементов интерфейса;
* обработка действий пользователя;
* маршрутизация между экранами.

#### **2) Сервисный слой**

* SupabaseService — работа с БД и хранилищем;
* ImageService — получением и обработка изображений.

#### **3) Модельный слой**

* классы Folder и Recipe;
* преобразование JSON ↔ модели;
* валидация входных данных.

#### **4) Backend слой (Supabase)**

* База данных Postgres;
* Supabase Auth;
* Supabase Storage;
* политики безопасности RLS.

# **14. Заключение**

Проектируемое приложение представляет собой удобное персональное пространство для хранения и систематизации кулинарных рецептов. Оно учитывает реальные потребности пользователей, позволяя им создавать тематические папки, работать с изображениями и отмечать избранные блюда. Использование Flutter и Supabase обеспечивает высокую скорость разработки, кроссплатформенность и безопасность данных. Архитектура приложения легко расширяема, что делает продукт перспективным для дальнейшего развития. Реализация описанных механизмов позволит создать современное и интуитивно понятное приложение, ориентированное на личные коллекции рецептов. Впоследствии планируется публикация приложения в RuStore.
